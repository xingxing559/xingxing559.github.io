    # 5. 基础回顾-关联表查询

    上一章，我们快速的过了一遍单表增删改查的写法，以及两种 Dao 开发的方式。通常我们都会选择 Mapper 动态代理的方式，所以以后所有的代码，如果没有特殊声明，我们都是使用 Mapper 动态代理的编码方式了。

    这一章我们来回顾一下关联表实体的查询，以及这里面涉及到的一些特殊机制。通过这三章的快速回顾，希望各位小伙伴能快速调动起自己可能尘封已久的回忆，为下面 MyBatis 的深入学习做好铺垫。

    # 1. 用户User的初始化

    编码之前，我们先初始化一个新的实体：用户，以此来形成 “部门 - 用户” 的一对多关联。

    # 1.1 表创建

    后续我们还会经常用到这两张表，所以这个用户表我打算设计的稍微复杂一点，方便后面我们讲解更多的东西。

    CREATE TABLE tbl_user (
  id varchar(32) NOT NULL,
  version int(11) NOT NULL DEFAULT '0',
  name varchar(32) NOT NULL,
  age int(3) DEFAULT NULL,
  birthday datetime DEFAULT NULL,
  department_id varchar(32) NOT NULL,
  sorder int(11) NOT NULL DEFAULT '1',
  deleted tinyint(1) NOT NULL DEFAULT '0',
  PRIMARY KEY (id)

    # 2. User对Department多对一

    关联多对一的查询，分两种情况：立即加载，延迟加载。我们先从立即加载开始吧。

    # 2.1 mapper.xml

    首先要写的是 mapper.xml ，这里面我们要自己定义一个 resultMap ，因为MyBatis 不像 Hibernate 那样是全自动 ORM ，对于实体类型它无法直接识别，需要我们去做手动映射。

    <?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.linkedbear.mybatis.mapper.UserMapper">
    <resultMap id="userMap" type="com.linkedbear.mybatis.entity.User">
        <id property="id" column="id"/>
        <result property="name" column="name"/>
        <result property="age" column="age"/>
        <result property="birthday" column="birthday"/>
        <association property="department" javaType="com.linkedbear.mybatis.entity.Department">
            <id property="id" column="department_id"/>
            <result property="name" column="department_name"/>
        </association>
    </resultMap>
</mapper>

        <select id="findAll" resultMap="userMap">
        select usr.*, dep.name as department_name

        left join tbl_department dep on usr.department_id = dep.id

    # 2.4 Department延迟加载

    如果需要实现像 Hibernate 那样的关联对象延迟加载，可以通过修改 resultMap ，使其延迟加载：

        <resultMap id="userlazy" type="com.linkedbear.mybatis.entity.User">
        <id property="id" column="id"/>
        <result property="name" column="name"/>
        <result property="age" column="age"/>
        <result property="birthday" column="birthday"/>
        <association property="department" javaType="com.linkedbear.mybatis.entity.Department"
                     select="com.linkedbear.mybatis.mapper.DepartmentMapper.findById" column="department_id"/>
    </resultMap>

    给 association 标签添加 select 和 column 属性，分别指定触发延迟加载时调用的 statement ，以及执行 statement 时传入的参数的列，就可以实现延迟加载了。

    相应的，再定义一个新的 statement ，并应用上面的 resultMap ：

        <select id="findAllLazy" resultMap="userlazy">
        select * from tbl_user
    </select>

    # 3.2 mapper.xml修改

    改完实体类，接下来还要修改 mapper.xml 由于这次需要在查出 部门 的同时，把用户信息也拉出来，所以必然要定义 resultMap 了。

        <resultMap id="department" type="com.linkedbear.mybatis.entity.Department">
        <id property="id" column="id"/>
        <result property="name" column="name"/>
        <result property="tel" column="tel"/>
        <collection property="users" ofType="com.linkedbear.mybatis.entity.User"
                    select="com.linkedbear.mybatis.mapper.UserMapper.findAllByDepartmentId" column="id"/>
    </resultMap>

        <select id="findAllByDepartmentId" parameterType="string" resultType="com.linkedbear.mybatis.entity.User">
        select * from tbl_user where department_id = #{departmentId}
