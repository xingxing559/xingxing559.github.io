    # 37. 二次封装-元编程思路优化CRUD处理

    OK ，经过上一章的设计，我们依靠 Provider 系列注解，配合动态 SQL ，完成了一个基本的单表 CRUD ，不过这里面有一些代码着实让我们感觉到不舒服：

    每次生成 SQL 都会走一次反射，且不说 jdk8 以后对反射的性能还有一些优化了，就这样每次都反射一波，最终得到的效果还基本一样，很明显没有必要每次都反射。那如何能做到不需要每次都反射呢？

    元编程！看过 SpringFramework 的源码，或者直接看过阿熊的 SpringFramework 小册的小伙伴，一定对 BeanDefinition 的设计深有体会吧！加载好一次后，将这个 bean 的一些基本信息都记录下来，回头用到的时候直接取。按照相同的设计思路，我们可以将这些实体模型类都只解析一次，保存到一个独立的空间，之后每次要生成 SQL 的时候，直接读这个实体模型类的元信息即可。

    OK ，按照这个思路，下面我们开始。

    本章代码位置：com.linkedbear.mybatis.encapsulate.b_metadata

    # 1. 元信息的定义？

    仔细观察一下，解析一个实体模型类，可以获得什么，以及我们需要哪些。

    • 表名
    • 主键字段、生成策略
    • 需要与数据库映射的属性、更新策略
    • 不需要与数据库映射的属性
    • 后续扩展的其他特性

    由这个分析，我们可以粗略的形成一个元信息的模型类：

    public class EntityMetadata {
    
    private Class<?> entityClass;
    
    private String tablename;
    
    private String idField;
    
    private String idColumn;
    
    private IdGenerator idGenerator;
    
    private List<FieldMetadata> fields = new ArrayList<>();
}
public class FieldMetadata {
    
    private String fieldName;
    
    private String columnName;
    
    private boolean primary = false;
    
    private boolean updateIfNull = false;
}

    按照这种提取方式，是不是我们所需要的实体模型类中的信息，都可以收集到这样一个 EntityMetadata 中呢？

    当然，只有元信息，没有一个中央管理者也是不行的，好歹得根据实体类的类型，找到对应的 EntityMetadata 吧，所以我们还需要一个元信息的容器：

    public enum EntityMetadataManager {
    INSTANCE;
    
    private ConcurrentMap<Class<?>, EntityMetadata> entityMetadataMap = new ConcurrentHashMap<>();
    
    public void registerMetadata(EntityMetadata metadata) {
        this.entityMetadataMap.put(metadata.getEntityClass(), metadata);
    }
    
    public EntityMetadata getMetadata(Class<?> entityClass) {
        return entityMetadataMap.get(entityClass);
    }
    
    public EntityMetadata getMetadataByMapper(Class<?> mapperClass) {
        return getMetadata(ClassUtils.getEntityClass(mapperClass));
    }
}

    怎么解析呢？其实很简单，就是把上一章我们在每个动态 SQL 创建时，解析实体类的逻辑，放到这里便可：

    protected void checkDaoConfig() {
    super.checkDaoConfig();
    Class<?> entityClass = ClassUtils.getEntityClass(this.getMapperInterface());
    EntityMetadata entityMetadata = new EntityMetadata(entityClass);
    entityMetadata.setTablename(resolveTablename(entityClass));

    ReflectionUtils.doWithFields(entityClass, field -> {
        // 列名的处理（约定大于配置）
        Column columnAnno = field.getAnnotation(Column.class);
        String column = columnAnno != null && StringUtils.hasText(columnAnno.value())
                ? columnAnno.value()
                : StringConvertUtils.camelCaseToUnderscore(field.getName());

        FieldMetadata fieldMetadata = new FieldMetadata(field.getName(), column);
        // 解析自定义注解
        if (field.isAnnotationPresent(Id.class)) {
            fieldMetadata.setPrimary(true);
            entityMetadata.setIdField(field.getName());
            entityMetadata.setIdColumn(column);
            entityMetadata.setIdGenerator(BeanUtils.instantiateClass(field.getAnnotation(Id.class).generateStrategy()));
        }
        if (columnAnno != null) {
            fieldMetadata.setUpdateIfNull(columnAnno.updateIfNull());
        }
        entityMetadata.getFields().add(fieldMetadata);
    }, field -> !field.isAnnotationPresent(Transient.class) && !Modifier.isStatic(field.getModifiers()));

    // 实体类元信息存放至统一容器
    EntityMetadataManager.INSTANCE.registerMetadata(entityMetadata);
}

private String resolveTablename(Class<?> entityClass) {
    if (entityClass.isAnnotationPresent(Table.class)) {
        return entityClass.getAnnotation(Table.class).value();
    }
    String className = entityClass.getName();
    return StringConvertUtils.camelCaseToUnderscore(Introspector.decapitalize(className));
}

    # 2.3 改造BaseProvider

    接下来的步骤就是改造 BaseProvider 了，我们只需要拿到 Mapper 接口，那后面什么也都能拿到了：

    public String findAll(Object params, ProviderContext providerContext) {
    EntityMetadata entityMetadata = EntityMetadataManager.INSTANCE.getMetadataByMapper(providerContext.getMapperType());
    String tablename = entityMetadata.getTablename();

    SQL sql = new SQL();
    sql.SELECT("*").FROM(tablename);
    return sql.toString();
}

public String insert(Object params, ProviderContext providerContext) {
    EntityMetadata entityMetadata = EntityMetadataManager.INSTANCE.getMetadata(providerContext.getMapperType());
    String tablename = entityMetadata.getTablename();
    List<FieldMetadata> fieldMetadatas = entityMetadata.getFields();
    String[] columns = fieldMetadatas.stream().map(FieldMetadata::getColumnName).toArray(String[]::new);
    String[] fields = fieldMetadatas.stream().map(fm -> "#{" + fm.getFieldName() + "}").toArray(String[]::new);
    // 给实体类生成id
    SystemMetaObject.forObject(params).setValue(entityMetadata.getIdField(), entityMetadata.getIdGenerator().next());

    SQL sql = new SQL();
    sql.INSERT_INTO(tablename).INTO_COLUMNS(columns).INTO_VALUES(fields);
    return sql.toString();
}

    # 2.4 BaseProvider的一个问题

    仔细观察一下此时的 BaseProvider ，我们会发现一个问题：如果我们需要扩展一个新的 delete 方法，需要我们修改 BaseProvider 的源码，从设计模式的角度上讲，这违背了开放封闭原则。为此，我们需要进一步优化 Provider 的写法。

    # 3. 重构BaseProvider

    既然要设计为符合开放封闭原则的结构，那么在一个类中编写多个方法，这种方式显然需要改进。

    # 3.1 Provider系列方法的共性

    我们可以先分析一下 BaseProvider 中目前的几个方法的特点。

    • 都定义了相同的入参
    • 都是相同的创建动态 SQL

    唯一不同的，是构建动态 SQL 的过程。

    很明显，这个改造的思路就出来了：我们可以使用模板方法模式，定义一个基础的 Provider 父类，由具体的子类决定如何构建 SQL 。

    # 3.2 代码重构

    OK ，思路有了，下面我们编码实现。

    首先来一个父类的 Provider ，这里面的逻辑不用很复杂，主要是留下一个用于构造 SQL 语句的模板方法即可：

    public abstract class AbstractProvider {
    
    public String invoke(Object params, ProviderContext context) {
        EntityMetadata entityMetadata = EntityMetadataManager.INSTANCE.getMetadataByMapper(context.getMapperType());
        return buildSql(entityMetadata, params);
    }
    
    protected abstract String buildSql(EntityMetadata entityMetadata, Object params);
}

    接下来，每个 Mapper 方法实现，只需要写一个一个的实现类即可：

    public class FindAll extends AbstractProvider {
    
    @Override
    protected String buildSql(EntityMetadata entityMetadata, Object params) {
        SQL sql = new SQL();
        sql.SELECT("*").FROM(entityMetadata.getTablename());
        return sql.toString();
    }
}
public class Insert extends AbstractProvider {
    
    @Override
    protected String buildSql(EntityMetadata entityMetadata, Object params) {
        String tablename = entityMetadata.getTablename();
        List<FieldMetadata> fieldMetadatas = entityMetadata.getFields();
        String[] columns = fieldMetadatas.stream().map(FieldMetadata::getColumnName).toArray(String[]::new);
        String[] fields = fieldMetadatas.stream().map(fm -> "#{" + fm.getFieldName() + "}").toArray(String[]::new);
        
        SystemMetaObject.forObject(params).setValue(entityMetadata.getIdField(), entityMetadata.getIdGenerator().next());
    
        SQL sql = new SQL();
        sql.INSERT_INTO(tablename).INTO_COLUMNS(columns).INTO_VALUES(fields);
        return sql.toString();
    }
}
